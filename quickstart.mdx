---
title: "Quickstart"
description: "Install the plugin, connect it to StandIn, and place a test Teams call."
---

Pick your runtime. Both paths get you to a live Teams call where your agent answers in voice.

## Prerequisites

- A working **OpenClaw** or **Hermes Agent** install (the host).
- A **StandIn** subscription ([standin.komaa.com](https://standin.komaa.com), free package) - the
  hosted media bridge that joins the Teams call and connects to your plugin. No worker to run.
- Your own **Microsoft Teams bot** (an Azure AD app) connected to StandIn - see [Teams app setup](/teams-app)
  to create and submit it.
- A realtime voice provider key (for **realtime** mode) - e.g. OpenAI or Azure OpenAI. Streaming mode
  uses your host's configured STT/TTS instead.

<Tabs>
  <Tab title="OpenClaw (npm)">
    <Steps>
      <Step title="Install">
        ```bash
        openclaw plugins install clawhub:@komaa/msteams-voice
        cd extensions/msteams-voice && pnpm install && pnpm build
        ```
      </Step>
      <Step title="Configure">
        Add a `msteams-voice` entry to your OpenClaw config - `sharedSecret` must match the value you set in StandIn:
        ```jsonc
        {
          "plugins": {
            "entries": {
              "msteams-voice": {
                "config": {
                  "enabled": true,
                  "mode": "realtime",
                  "port": 9442,
                  "path": "/voice/msteams/stream",
                  "sharedSecret": "<same secret as in your StandIn dashboard>",
                  "inboundPolicy": "allowlist",
                  "allowFrom": ["<caller AAD object id>"],
                  "realtime": {
                    "provider": "openai",
                    "providers": {
                      "openai": { "apiKey": "<key>", "model": "gpt-realtime" }
                    }
                  }
                }
              }
            }
          }
        }
        ```
        Full reference → [OpenClaw configuration](/openclaw/configuration).
      </Step>
      <Step title="Run + call">
        Start the gateway (`openclaw gateway run`), then in your StandIn dashboard register your
        plugin's WebSocket (`ws://<host>:9442/voice/msteams/stream`) and the matching secret. Place a Teams call to your bot.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Hermes (pip)">
    <Steps>
      <Step title="Install">
        Into the **same Python environment as Hermes**:
        ```bash
        uv pip install --python /path/to/hermes/venv/bin/python hermes-plugin-teams-voice
        hermes plugins enable teams_voice
        ```
      </Step>
      <Step title="Configure">
        In `~/.hermes/config.yaml` under `plugins.entries.teams_voice.config` (secrets in `.env`):
        ```yaml
        plugins:
          enabled: [teams_voice]
          entries:
            teams_voice:
              config:
                shared_secret: ${TEAMS_VOICE_SHARED_SECRET}   # must match the secret set in StandIn
                host: 127.0.0.1
                port: 8443
                realtime:
                  backend: azure
                  azure_endpoint: https://<your-azure-resource>.cognitiveservices.azure.com
                  azure_deployment: gpt-realtime
                  api_key: ${AZURE_FOUNDRY_API_KEY}
        ```
        Full reference → [Hermes configuration](/hermes/configuration).
      </Step>
      <Step title="Run + call">
        ```bash
        hermes teams-voice serve --handler realtime
        ```
        In your StandIn dashboard, register your plugin's WebSocket (`ws://<host>:8443/voice/msteams/stream`)
        and the matching secret, then place a Teams call to your bot.
      </Step>
    </Steps>
  </Tab>
</Tabs>

<Warning>
  **StandIn connects to your plugin from the internet**, so its WebSocket must be reachable - expose it
  via a public URL or a tunnel (e.g. Tailscale), not loopback-only. Default ports **9442** (OpenClaw) /
  **8443** (Hermes); path `/voice/msteams/stream`. A `sharedSecret` mismatch fails the HMAC handshake
  **silently** - the most common first-call issue.
</Warning>

<Tip>
  Streaming mode (no realtime key needed) is a one-line change - see [Modes](/concepts/modes).
</Tip>
