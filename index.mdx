**Komaa Teams Voice** lets your agent **join a Microsoft Teams call as a real participant** — it
listens, speaks, sees shared video, and drives an on-screen avatar. It ships as a **standalone plugin
you install on top of your existing agent** (no fork), in two flavors for the two runtimes:

<CardGroup cols={2}>
  <Card title="OpenClaw plugin" icon="npm" href="/openclaw/installation">
    `@komaa/msteams-voice` — a TypeScript plugin for **OpenClaw**, installed via npm / `openclaw plugins install`.
  </Card>
  <Card title="Hermes plugin" icon="python" href="/hermes/installation">
    `hermes-plugin-teams-voice` — a Python plugin for **Hermes Agent**, installed via `pip` / `uv`.
  </Card>
</CardGroup>

Both share the same Conversational Video Interface (CVI) and the same call architecture — only the
runtime and packaging differ.

## What it does

- 🎙️ **Realtime speech-to-speech** (OpenAI / Azure) **or** streaming **STT → agent → TTS** (any provider)
- 👁️ **Vision** — the agent can look at screen-share / camera frames, with a per-minute budget
- 🧑‍🤝‍🧑 **Group-call etiquette** — speaks only when addressed by a wake phrase
- 📞 **Outbound call-backs** · 📝 **meeting recap / minutes** · 🌐 **bilingual EN/AR** · ⌨️ **DTMF**
- 🪄 **Avatar driver cues** — expression, viseme lip-sync, show-to-caller
- 🔐 **HMAC-authenticated media bridge** + caller allowlist + recording-status gate

## How it fits together

A small **media worker** dials an HMAC-authenticated WebSocket that the plugin hosts; the plugin runs
the dialogue, perception, and avatar cues. See [Architecture](/concepts/architecture).
